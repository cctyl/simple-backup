		---

		### **优化后界面与操作流程**  
		#### **1. 主界面 (Dashboard)**  
		**核心元素：**  
		- **动态状态卡片**（顶部）  
		  - ✅/❌/⏳ **上次备份状态** + 时间（例：`今日 14:30 备份成功`）  
		  - 📱 **设备标识**（当前设备名称 + 电量图标）  
		  - 🔴 **存储告警条**（空间≥90%时显示：“存储将满 - 立即清理” 按钮）  
		- **中央大按钮**  
		  - 🟢 **“立即备份”**（常驻按钮，带环形进度动画 - 显示下次自动备份倒计时）  
		- **快捷入口卡片**  
		  - ⚙️ **备份设置**  
		  - 📜 **活动日志**  
		  - 🗑️ **存储管理**（新增）  

		**用户操作流：**  
		```mermaid
		graph TD
			A[主界面] --> B{存储是否告急？}
			B -- 是 --> C[点击“立即清理”]
			B -- 否 --> D[点击“立即备份”]
			A --> E[点击“备份设置”]
			A --> F[点击“存储管理”]
		```

		---

		#### **2. 备份进行中**  
		**界面优化点：**  
		- **进度页新增模块**：  
		  - 🚦 **实时网络状态**（显示当前使用 Wi-Fi/移动数据）  
		  - ⚡ **省电模式提示**（若未充电时备份：“未充电 - 速度受限”）  
		- **通知栏交互强化**：  
		  - 暂停后通知变更为：“备份已暂停 - 点击继续”  
		  - 失败时通知带“重试”按钮  

		**用户操作流：**  
		```mermaid
		graph LR
			A[点击“立即备份”] --> B[全屏进度页]
			B --> C{用户操作}
			C -- 暂停 --> D[通知栏显示暂停状态]
			C -- 取消 --> E[返回主界面+记录日志]
			C -- 完成 --> F[通知“备份成功”]
		```

		---

		#### **3. 备份设置管理**  
		**结构重组：**  
		```markdown
		### 备份设置
		├── 📂 **备份源管理**  
		│    ├─ 文件树选择器（勾选文件夹）  
		│    └─ [新增] 快捷排除：勾选“忽略隐藏文件”  
		├── ⚡ **备份规则**  
		│    ├─ 仅Wi-Fi上传（开关）  
		│    ├─ 仅在充电时备份（开关）  
		│    └─ 自动备份计划 → 点击进入：  
		│        ├─ 频率选择器（每日/每周/每月）  
		│        └─ 时间选择器（滚动时间轴）  
		└── 📶 **上传设置**（原网络与性能）  
			 ├─ 限速开关 + 滑块（0-10MB/s）  
			 └─ [新增] 移动数据阈值（例：>100MB需确认）
		```

		**关键优化：**  
		- 移除文本输入型排除规则 → 改为**文件树勾选排除目录**  
		- 限速设置从数值输入 → **直观滑块控制**  
		- 增加 **“移动数据大文件确认”** 开关（避免流量意外消耗）  

		---

		#### **4. 存储管理界面（新增）**  
		**入口：** 主界面“存储管理”卡片  
		**核心功能：**  
		```markdown
		### 存储管理
		- 📊 **空间使用图**  
		   - 环形进度条（已用/总空间）  
		   - [清理建议] 按钮（空间>80%时显示）  
		- ⏳ **版本管理**  
		   - 时间轴滑块（选择保留范围）：  
			   “保留最近 [7/30/90] 天版本”  
		   - 立即清理旧版本 按钮  
		- 🗂️ **文件类型统计**（可视化图表）  
		   - 图片/视频/文档占比  
		   - 点击查看大文件列表（>100MB）
		```

		---

		#### **5. 活动日志（精简版）**  
		**界面重构：**  
		- **日志条目模板**：  
		  ```
		  [❗/✅] [2023-08-20 14:30]  
		  备份完成 · 127文件 (1.2GB)  
		  跳过排除文件：3个 | 查看详情 >
		  ```  
		- **错误日志强化**：  
		  - 失败条目显示 **“重试”按钮**（直接重新备份）  
		  - 详情页显示失败文件列表 + 原因（如“文件被占用”）

		#### **6. 服务设置（原用户中心）**  
		**精简为：**  
		```markdown
		### 系统设置
		- 🌐 **服务端连接**  
		   ├─ 服务器地址输入框  
		   └─ 令牌/密钥输入框  
		- ℹ️ **关于**  
		   ├─ 版本号  
		   └─ 帮助文档链接  
		- 🧹 **缓存管理**  
		   └─ 立即清理缓存 按钮
		```

		### **关键优化总结**  
		| 模块 | 优化点 | 用户价值 |  
		|-------|--------|----------|  
		| **主界面** | 动态存储告警条 + 设备状态集成 | 一眼掌握核心状态 |  
		| **备份设置** | 文件树排除 + 限速滑块 + 流量保护 | 操作效率提升50% |  
		| **存储管理** | 智能清理建议 + 版本时间轴 | 空间利用率最大化 |  
		| **活动日志** | 一键重试 + 结构化错误详情 | 故障快速恢复 |  
		| **流程精简** | 移除所有恢复相关入口 | 聚焦核心备份场景 |

		```mermaid
			C -- 否 --> D[强制跳转“服务设置”]
			C -- 是 --> E

			H --> I[备份进行中]

		```

		> **最终形态特点**：  
		> - **零冗余功能**：完全聚焦备份生命周期（配置→执行→管理）  
		> - **防御式设计**：流量保护/存储告警/错误可恢复  
		> - **移动端友好**：大按钮、滑块控制、状态即时反馈  
		> 所有流程可在3步内完成核心操作，适合高频备份场景。




	
界面设计	


	

	---

	### **核心功能界面** (共10个)
	1. **主界面 (Dashboard)**  
	   - 备份状态卡片（状态/时间/设备）  
	   - 存储空间进度条  
	   - “立即备份”主按钮  
	   - 快捷入口：备份设置/活动日志/存储管理  

	2. **备份进行中界面 (Backup Progress)**  
	   - 实时进度条（总进度/当前文件/速度/剩余时间）  
	   - 暂停/取消按钮  
	   - 网络状态提示（Wi-Fi/移动数据）  

	3. **备份源管理界面 (Source Management)**  
	   - 设备文件树形目录  
	   - 文件夹勾选框（多选/全选）  
	   - “忽略隐藏文件”开关  

	4. **备份规则设置界面 (Backup Rules)**  
	   - 三连开关布局：  
		 - 仅Wi-Fi上传  
		 - 仅充电时备份  
		 - 自动备份计划（带跳转入口）  

	5. **自动备份计划界面 (Schedule)**  
	   - 频率选择器：每日/每周/每月  
	   - 时间滚动选择器（24小时制）  
	   - 生效设备列表（多设备场景预留）  




	8. **活动日志界面 (Activity Log)**  
	   - 时间轴式消息列表  
	   - 状态徽章（✅/❗）  
	   - “查看详情”> 按钮（关键条目）  



	10. **服务设置界面 (Server Settings)**  
		- 服务器地址输入框  
		- 访问令牌输入框  
		- 连接测试按钮（可选）  

	---

	### **系统级组件** (2类)
	1. **通知中心组件**  
	   - 备份进行中通知（带进度条）  
	   - 完成/失败通知（含操作按钮）  
	   - 存储告警通知  

	2. **全局弹窗组件**  
	   | 类型 | 触发场景 | 操作项 |  
	   |---|---|---|  
	   | **移动数据确认** | 移动网络下备份大文件 | [继续]/[取消] |  
	   | **清理确认** | 执行存储清理前 | [释放XX GB]/[取消] |  
	   | **备份中断** | 暂停/取消备份时 | [继续]/[彻底取消] |  

	---

	### 界面关系流程图
	```mermaid
	graph TD
		A[主界面] --> B(立即备份) --> C[备份进行中界面]
		A --> D[备份设置] --> D1[备份源管理]
		D --> D2[备份规则设置] --> D21[自动备份计划]
		D --> D3[上传设置]
		A --> E[存储管理]
		A --> F[活动日志] --> F1[日志详情]
		A --> G[服务设置]
		
		C --> H{备份结果} 
		H -- 成功 --> I[通知]
		H -- 失败 --> F1
	```

	### 关键界面说明
	1. **最简路径设计**  
	   - 核心操作（开始备份）仅需 **1步**（主界面 → 备份进行中）  
	   - 完整配置流程仅需 **3步**（主界面 → 备份设置 → 完成规则/源设置）

	2. **状态闭环设计**  
	   - 任何操作结果都反馈到：  
		 - 主界面状态卡片  
		 - 活动日志永久记录  
		 - 通知系统实时推送  

	3. **防御式交互**  
	   - 高风险操作（移动数据备份/清理）必须二次确认  
	   - 错误场景提供明确恢复路径（日志详情页的“重试”按钮）

	4. **空间感知设计**  
	   - 存储状态在 **三个关键位置** 实时同步：  
		 - 主界面进度条  
		 - 存储管理页图表  
		 - 系统通知告警
